# EBM

A component to encapsulate the running of EBM measures (both HEDIS and non-HEDIS)

## Development Guidelines

### Branching Stragegy

This repository utilizes full [git-flow](https://datasift.github.io/gitflow/IntroducingGitFlow.html)

### Program Headers

Each program should have an appropriate header on the file using the appropriate template from [here](https://indy-github.milliman.com/PRM/analytics-pipeline/tree/develop/000_Templates/QRM_Header_Templates/).

### Continuous integration

CI has been enabled for this repository. Developers are encouraged to submit unit tests with any library code. A base level of test coverage is required.

### Releases

For documentation purposes, each version of code used to make client deliverables should be tagged and documented as a [release](https://help.github.com/articles/creating-releases/).
